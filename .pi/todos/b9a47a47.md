## Changes Made

### 1. Model Updates (`internal/models/test_recording.go`)
- Added `SourceType` field to `TestRecording` struct - indicates origin: "manual" | "test_scenario"
- Added `SourceID` field to `TestRecording` struct - links to parent test scenario ID when applicable
- Added same fields to `RecordingSummary` struct for list responses

### 2. Handler Updates (`handlers/recording.go`)
- Updated `SaveRecording` to set `SourceType = "manual"` by default for user recordings
- Updated `ListRecordings` to support filtering by `source_type` query parameter

### 3. Test Scenario Handler (`handlers/test_scenario.go`)
- Updated test scenario generation to set `SourceType = "test_scenario"` and `SourceID = scenario.ID` for auto-generated recordings

### 4. Agent Tools Updates
- `agent/tools_recording.go`: Updated RecordingSummary conversion to include SourceType and SourceID
- `agent/tool_executor.go`: Updated RecordingSummary conversion to include SourceType and SourceID
- `agent/tools_test_recording.go`: 
  - Updated `saveTestRecording` to set source fields
  - Updated `buildRecordingSteps` to set source fields
  - Updated `agentGenerateSingleRecording` signature to accept scenarioID and set source fields

## API Changes

### New Query Parameter for List Recordings
`GET /api/recordings?source_type=manual|test_scenario`

### Response Fields
All recording responses now include:
- `source_type`: "manual" | "test_scenario" | "" (legacy)
- `source_id`: ID of parent test scenario (if applicable)
